\# PRD：跨平台实时双向语音翻译应用



\## 1. 产品概述



\### 1.1 产品名称



暂定名称：\*\*AI Translator Audio Gateway\*\*



中文名称：\*\*实时语音翻译音频网关\*\*



\### 1.2 产品定位



本产品是一款运行在 \*\*macOS 与 Windows\*\* 系统上的实时双向语音翻译工具。



产品通过在操作系统中提供虚拟音频设备，将本地用户与第三方语音聊天工具之间的音频流进行实时翻译，从而实现不同语言用户之间的自然语音沟通。



本产品不替代 Zoom、Teams、Discord、微信、企业微信等第三方语音工具，而是作为系统级音频中间层接入这些工具。



\### 1.3 核心场景



用户 A 在电脑上通过第三方语音聊天工具与用户 B 通话。



用户 A 使用中文，用户 B 使用英文。



应用启动后：



\* 用户 A 说中文，用户 B 听到英文。

\* 用户 B 说英文，用户 A 听到中文。

\* 第三方语音软件无需改造，只需要选择本应用提供的虚拟麦克风和虚拟扬声器。



\---



\## 2. 背景与问题



\### 2.1 当前问题



跨语言语音沟通存在以下问题：



1\. 用户需要依赖人工翻译或手动输入翻译工具。

2\. 第三方会议软件自带翻译能力有限，且不同软件能力不一致。

3\. 实时语音翻译通常只能在特定会议系统内使用，无法覆盖所有语音聊天工具。

4\. 普通翻译工具无法直接接入系统音频链路，用户体验割裂。

5\. 用户希望保留原有语音聊天工具，但获得实时同传能力。



\### 2.2 产品机会



通过虚拟音频设备，本产品可以在系统层面拦截和注入音频流，实现对第三方语音工具的无侵入接入。



用户只需要在第三方语音软件中选择本应用提供的虚拟设备，即可获得实时双向语音翻译能力。

\## 3. 产品目标



\### 3.1 业务目标



1\. 打通 macOS 与 Windows 上的实时双向语音翻译闭环。

2\. 支持主流第三方语音聊天工具。

3\. 降低跨语言沟通门槛。

4\. 为后续多人会议翻译、字幕翻译、会议纪要等能力打基础。



\### 3.2 用户目标



1\. 用户无需更换现有语音聊天工具。

2\. 用户只需简单配置虚拟麦克风和虚拟扬声器。

3\. 用户可以用自己的母语与不同语言的人实时通话。

4\. 翻译延迟在可接受范围内，不影响基本对话节奏。

5\. 用户可以清楚知道当前音频链路是否正常工作。



\### 3.3 技术目标



1\. 实现双向实时音频采集、翻译和播放。

2\. 支持虚拟 MIC 与虚拟 Speaker。

3\. 支持 Gemini Live Translate 实时语音翻译能力。

4\. 保证音频链路稳定，避免回声、串音和音频循环。

5\. 支持断线重连、设备热插拔、异常恢复。

\## 5. 使用场景



\### 5.1 核心场景：一对一中英双向语音聊天



\#### 场景描述



用户 A 使用中文，用户 B 使用英文。



用户 A 使用第三方语音聊天工具与用户 B 通话。



用户 A 在本地启动实时翻译应用，并在第三方语音聊天工具中配置：



| 配置项 | 选择                    |

| --- | --------------------- |

| 麦克风 | AI Translator Mic     |

| 扬声器 | AI Translator Speaker |



\#### 预期效果



\* 用户 A 说中文，系统翻译成英文后发送给用户 B。

\* 用户 B 说英文，系统翻译成中文后播放给用户 A。

\* 用户 B 不需要安装本应用。

\* 第三方语音软件不需要做任何改造。



\---



\### 5.2 场景二：只翻译自己说的话



\#### 场景描述



用户 A 希望自己说中文，对方听英文，但用户 A 可以直接听懂对方英文。



\#### 预期效果



\* 开启上行翻译：中文 → 英文。

\* 关闭下行翻译。

\* 用户 B 听到翻译后的英文。

\* 用户 A 直接听到用户 B 的原始英文。



\---



\### 5.3 场景三：只翻译对方说的话



\#### 场景描述



用户 A 可以说英文，但听英文有困难，希望对方英文被翻译成中文。



\#### 预期效果



\* 关闭上行翻译。

\* 开启下行翻译：英文 → 中文。

\* 用户 A 说的原始英文直接传给用户 B。

\* 用户 B 说的英文被翻译成中文后播放给用户 A。



\---



\### 5.4 场景四：会议前设备检查



\#### 场景描述



用户在正式通话前，希望检查麦克风、扬声器、虚拟设备和翻译链路是否正常。



\#### 预期效果



系统提供检测流程：



1\. 检测物理麦克风是否可用。

2\. 检测物理扬声器或耳机是否可用。

3\. 检测虚拟麦克风是否安装。

4\. 检测虚拟扬声器是否安装。

5\. 检测 Gemini 连接是否正常。

6\. 播放测试语音并返回测试结果。

\## 8. 功能需求



\## 8.1 设备管理



\### 8.1.1 物理麦克风选择



\#### 需求说明



用户可以选择本机可用的物理麦克风。



系统需要支持：



\* 展示系统当前可用输入设备。

\* 标识默认麦克风。

\* 支持设备热插拔。

\* 当前设备不可用时，提示用户重新选择。

\* 支持测试麦克风音量。



\#### 验收标准



\* 插入新麦克风后，应用可以刷新设备列表。

\* 拔出当前使用的麦克风后，应用提示设备不可用。

\* 用户说话时，界面可以显示输入音量变化。



\---



\### 8.1.2 物理扬声器或耳机选择



\#### 需求说明



用户可以选择翻译后音频播放到哪个物理输出设备。



系统需要支持：



\* 展示系统当前可用输出设备。

\* 标识默认扬声器。

\* 推荐用户使用耳机，避免回声。

\* 支持播放测试音。



\#### 验收标准



\* 用户可以听到测试音。

\* 切换输出设备后，翻译音频从新设备播放。

\* 未检测到可用输出设备时，给出明确提示。



\---



\### 8.1.3 虚拟麦克风检测



\#### 需求说明



应用需要检测系统中是否存在虚拟麦克风设备。



设备名称建议：



```text

AI Translator Mic

```



MVP 阶段可以依赖第三方虚拟音频设备：



| 平台      | MVP 虚拟音频方案             |

| ------- | ---------------------- |

| macOS   | BlackHole              |

| Windows | VB-CABLE / VoiceMeeter |



产品化阶段使用自有虚拟音频驱动。



\#### 验收标准



\* 应用可以识别可用虚拟输入设备。

\* 第三方语音软件中可以选择该虚拟麦克风。

\* 应用输出到虚拟麦克风后，第三方语音软件可以采集到声音。



\---



\### 8.1.4 虚拟扬声器检测



\#### 需求说明



应用需要检测系统中是否存在虚拟扬声器设备。



设备名称建议：



```text

AI Translator Speaker

```



\#### 验收标准



\* 应用可以识别可用虚拟输出设备。

\* 第三方语音软件可以将声音输出到该虚拟扬声器。

\* 应用可以从该虚拟扬声器采集远端用户声音。



\---



\## 8.2 翻译模式



\### 8.2.1 双向翻译模式



\#### 需求说明



用户可以开启双向翻译。



默认场景：



```text

中文 → 英文

英文 → 中文

```



系统需要支持：



\* 用户 A 说中文，用户 B 听到英文。

\* 用户 B 说英文，用户 A 听到中文。

\* 上行和下行翻译使用独立音频链路。

\* 上行和下行翻译可以独立开关。



\#### 验收标准



\* 用户本地说中文时，第三方语音工具收到英文语音。

\* 对方说英文时，用户本地听到中文语音。

\* 两条链路互不干扰，不产生循环播放。



\---



\### 8.2.2 单向上行翻译



\#### 需求说明



用户可以只开启本地说话翻译。



示例：



```text

中文 → 英文

```



\#### 验收标准



\* 本地用户说中文，对方听到英文。

\* 对方说英文，本地用户直接听原始英文或按用户配置处理。

\* 下行 Gemini 连接不启动，减少成本。



\---



\### 8.2.3 单向下行翻译



\#### 需求说明



用户可以只开启对方说话翻译。



示例：



```text

英文 → 中文

```



\#### 验收标准



\* 本地用户说话不被翻译。

\* 对方说英文，本地用户听到中文。

\* 上行 Gemini 连接不启动，减少成本。



\---



\### 8.2.4 语言选择



\#### 需求说明



MVP 版本支持：



\* 中文简体

\* 英文



后续版本支持更多语言。



系统需要支持：



\* 用户可以选择本地语言。

\* 用户可以选择对方语言。

\* 系统根据语言配置自动创建对应翻译链路。



\#### 验收标准



\* 用户可以在设置中切换语言方向。

\* 切换语言后，新通话链路按新语言配置生效。

\* 语言切换过程中不导致应用崩溃。



\---



\## 8.3 实时音频处理



\### 8.3.1 麦克风音频采集



\#### 需求说明



应用需要采集用户真实麦克风输入。



系统需要支持：



\* 采集物理麦克风音频。

\* 转换为 Gemini Live Translate 要求的音频格式。

\* 按实时音频块发送。

\* 支持静音控制。



\#### 验收标准



\* 用户说话后，应用可以采集到音频。

\* 静音后，不再向翻译服务发送音频。

\* 取消静音后，音频发送恢复。



\---



\### 8.3.2 虚拟扬声器音频采集



\#### 需求说明



应用需要从虚拟扬声器采集第三方语音工具输出的远端声音。



系统需要支持：



\* 第三方语音工具将输出设备设置为虚拟扬声器。

\* 应用从虚拟扬声器采集音频。

\* 将采集到的远端语音发送到翻译链路。



\#### 验收标准



\* 第三方语音工具播放声音时，应用可以显示输入音量。

\* 应用可以将远端语音翻译后播放到物理耳机。

\* 用户不会同时听到原文和译文，除非主动开启原文监听。



\---



\### 8.3.3 重采样与格式转换



\#### 需求说明



Gemini Live Translate 输入要求：



```text

16kHz / 16-bit / mono / PCM

```



Gemini Live Translate 输出要求：



```text

24kHz / 16-bit / mono / PCM

```



应用需要完成：



\* 系统音频格式转换为 Gemini 输入格式。

\* Gemini 输出格式转换为系统播放格式。

\* 单声道与双声道转换。

\* 音量归一化。



\#### 验收标准



\* 不同系统默认采样率下，应用都可以正常工作。

\* 翻译输出没有明显爆音、破音、变速或音高异常。

\* 切换设备后音频格式可重新协商。



\---



\### 8.3.4 VAD 语音活动检测



\#### 需求说明



应用需要支持基础语音活动检测。



系统需要支持：



\* 检测用户是否正在说话。

\* 静音或背景噪声时，减少向翻译服务发送无效音频。

\* 降低成本和无效翻译。



\#### 验收标准



\* 用户不说话时，应用不持续触发翻译输出。

\* 背景轻微噪音不会频繁触发翻译。

\* 用户开始说话后，翻译链路可以及时恢复。



\---



\### 8.3.5 降噪与自动增益



\#### 需求说明



应用需要支持基础降噪和自动增益能力。



系统需要支持：



\* 降低环境噪声对翻译质量的影响。

\* 保证输入音量稳定。

\* 避免用户声音过小导致识别失败。



\#### 验收标准



\* 在正常办公环境中，翻译结果可理解。

\* 输入音量过低时，应用给出提示。

\* 输入音量过高时，应用提示可能出现爆音。



\---



\## 8.4 Gemini Live Translate 接入



\### 8.4.1 实时连接管理



\#### 需求说明



应用需要维护 Gemini Live Translate 实时连接。



系统需要支持：



\* 双向翻译模式下，创建两个独立翻译 Session。

\* 上行 Session：本地语言 → 对方语言。

\* 下行 Session：对方语言 → 本地语言。

\* 支持连接状态展示。

\* 支持断线重连。



\#### 验收标准



\* 启动翻译后，连接状态显示为已连接。

\* 网络断开时，状态显示异常。

\* 网络恢复后，应用可以自动重连或提示用户重试。

\* 重连后翻译链路恢复正常。



\---



\### 8.4.2 音频流发送



\#### 需求说明



应用需要将采集音频持续发送至 Gemini Live Translate。



系统需要支持：



\* 音频按固定时间片发送。

\* 优先采用 100ms 音频块。

\* 保证发送顺序。

\* 异常时丢弃过期音频，避免延迟堆积。



\#### 验收标准



\* 长时间通话过程中，音频不会无限堆积。

\* 网络抖动时，应用不会持续播放严重滞后的翻译内容。

\* 停止翻译后，不再发送音频。



\---



\### 8.4.3 翻译音频接收



\#### 需求说明



应用需要接收 Gemini 返回的翻译后语音。



系统需要支持：



\* 接收翻译音频流。

\* 进入播放缓冲区。

\* 上行翻译结果写入虚拟麦克风。

\* 下行翻译结果播放到物理耳机或扬声器。



\#### 验收标准



\* 翻译语音能够连续播放。

\* 不出现长时间卡顿。

\* 不出现明显音频错位。



\---



\### 8.4.4 API Key 与 Token 安全



\#### 需求说明



MVP 阶段可支持用户本地配置 API Key。



产品化阶段需要支持服务端临时 Token。



系统需要支持：



\* 不在客户端内置固定 API Key。

\* API Key 本地加密存储。

\* 企业版或商业版通过服务端下发临时 Token。

\* 服务端负责额度控制、账号鉴权、调用统计。



\#### 验收标准



\* 客户端安装包中不能明文包含 API Key。

\* 用户输入的 API Key 不明文展示。

\* 删除账号或退出登录后，本地凭据被清理。

\## 8.5 音频输出与路由



\### 8.5.1 上行输出到虚拟麦克风



\#### 需求说明



应用需要将本地用户翻译后的语音输出到虚拟麦克风。



流程：



```text

用户说中文

&#x20; → Gemini 返回英文语音

&#x20; → 应用将英文语音写入虚拟麦克风

&#x20; → 第三方语音软件将虚拟麦克风作为输入

&#x20; → 用户 B 听到英文

```



\#### 验收标准



\* 第三方语音软件麦克风测试中可以检测到翻译后的英文语音。

\* 对方听到的是英文翻译音频，而不是本地中文原声。

\* 本地中文原声不会被直接发送给对方。



\---



\### 8.5.2 下行输出到物理耳机或扬声器



\#### 需求说明



应用需要将对方语音翻译后的音频播放给本地用户。



流程：



```text

第三方语音软件输出英文到虚拟扬声器

&#x20; → 应用采集英文

&#x20; → Gemini 返回中文语音

&#x20; → 应用播放中文语音到物理耳机或扬声器

&#x20; → 用户 A 听到中文

```



\#### 验收标准



\* 用户听到中文翻译音频。

\* 默认不播放英文原音。

\* 用户可选择是否打开原音监听。



\---



\### 8.5.3 音频监听模式



\#### 需求说明



用户可以选择是否监听原始音频。



可选项：



1\. 只听译文。

2\. 原文小音量 + 译文主音量。

3\. 左耳原文，右耳译文。

4\. 只听原文，暂停译文。



MVP 默认只支持：



```text

只听译文

```

\## 8.8 异常处理



\### 8.8.1 麦克风权限异常



\#### 触发条件



\* 用户未授权麦克风权限。

\* 系统阻止应用访问麦克风。



\#### 提示文案



```text

无法访问麦克风。请在系统设置中允许 AI Translator 访问麦克风。

```



\#### 处理方式



\* 提供打开系统设置入口。

\* 授权后支持重新检测。



\---



\### 8.8.2 虚拟设备缺失



\#### 触发条件



\* 未检测到虚拟麦克风。

\* 未检测到虚拟扬声器。



\#### 提示文案



```text

未检测到虚拟音频设备。请先安装或启用虚拟音频设备。

```



\#### 处理方式



\* MVP 阶段引导安装 BlackHole 或 VB-CABLE。

\* 产品化阶段引导安装本应用虚拟驱动。

\### 8.8.3 Gemini 连接失败



\#### 触发条件



\* API Key 无效。

\* 网络不可用。

\* 服务端返回错误。

\* Token 过期。



\#### 提示文案



```text

翻译服务连接失败，请检查网络、账号或 API Key 配置。

```



\#### 处理方式



\* 支持重新连接。

\* 展示错误类型。

\* 提供日志导出。



\---



\### 8.8.4 音频循环风险



\#### 触发条件



\* 虚拟麦克风和虚拟扬声器配置错误。

\* 应用检测到输出音频再次进入输入链路。

\* 输入音量持续异常。



\#### 提示文案



```text

检测到可能存在音频循环，请检查第三方语音软件的麦克风和扬声器配置。

```



\#### 处理方式



\* 自动暂停翻译。

\* 提供配置检查说明。

\* 引导用户重新选择设备。

\## 9. 非功能需求



\### 9.1 性能需求



| 指标       | 要求                  |

| -------- | ------------------- |

| 应用启动时间   | 3 秒内进入主界面           |

| CPU 占用   | 普通通话场景下保持可接受        |

| 内存占用     | MVP 阶段建议小于 500MB    |



\### 9.2 稳定性需求



1\. 支持连续通话 60 分钟不崩溃。

2\. 网络中断后可以恢复。

3\. 音频设备切换后应用不崩溃。

4\. Gemini 连接异常时应用可恢复。

5\. 第三方语音软件重启后，应用可继续使用。



\### 9.3 兼容性需求



首批支持系统：



| 系统      | 版本要求                    |

| ------- | ----------------------- |

| macOS   | macOS 13 及以上            |

| Windows | Windows 10 / Windows 11 |



首批兼容软件：



| 软件              | 优先级 |

| --------------- | --- |

| Zoom            | P0  |

| Microsoft Teams | P0  |

| Discord         | P1  |

| 微信              | P1  |

| 企业微信            | P1  |

| Google Meet     | P1  |



\### 9.4 安全与隐私需求



1\. 用户必须明确授权麦克风访问。

2\. 应用需要明确提示音频会被发送到云端翻译服务。

3\. 默认不保存通话录音。

4\. 默认不保存翻译后的语音。

5\. API Key 不允许明文存储。

6\. 日志中不得记录原始音频内容。

7\. 用户可以一键清除本地配置和凭据。

\## 13. 风险与应对



\### 13.1 翻译延迟过高



\#### 风险说明



实时语音翻译天然存在延迟，可能影响对话节奏。



\#### 应对方案



\* 使用低延迟音频缓冲。

\* 按 100ms 音频块发送。

\* 提供低延迟模式和稳定模式。

\* 避免本地音频队列无限堆积。



\---



\### 13.2 音频回声和循环



\#### 风险说明



如果用户未正确配置第三方语音软件，可能出现回声、串音或循环播放。



\#### 应对方案



\* 首次配置时强制做设备检测。

\* 推荐用户使用耳机。

\* 检测异常音频循环后自动暂停。

\* 提供配置修复向导。



\---



\### 13.3 虚拟音频驱动开发成本高



\#### 风险说明



macOS 和 Windows 自研虚拟音频驱动成本高，兼容性要求高。



\#### 应对方案



\* MVP 阶段优先使用 BlackHole、VB-CABLE 等成熟方案。

\* 产品验证后再考虑自研或授权虚拟音频驱动。

\* 驱动作为独立里程碑，不阻塞核心翻译能力验证。



\---



\### 13.4 第三方软件兼容性差异



\#### 风险说明



不同语音软件对虚拟音频设备的识别和处理方式不同。



\#### 应对方案



\* 建立兼容性测试矩阵。

\* 优先支持 Zoom、Teams。

\* 对每个软件提供独立配置说明。

\* 收集用户诊断日志。



\---



\### 13.5 隐私合规风险



\#### 风险说明



实时语音需要发送到云端翻译服务，涉及用户隐私。



\#### 应对方案



\* 首次使用明确告知用户。

\* 默认不保存录音。

\* 默认不保存翻译音频。

\* 日志不记录音频内容。

\* 支持一键清理本地数据。

\* 企业版支持更严格的数据策略说明。


